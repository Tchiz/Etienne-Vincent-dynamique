Laravel 4 : chapitre 7 : Un exemple de site statique

Publié le 19 janvier 2013 par bestmomo

Avec tout ce que nous avons vu jusqu’à présent il est tout à fait possible de réaliser un petit moteur de site statique et il est grand temps de mettre un peu en pratique tous ces éléments. Commençons par réaliser le template HTML de la page. Personnellement je commence par réaliser dans un premier une page HTML pour régler tous les aspects stylistiques. J’utilise Bootstrap de Twitter pour simplifier la mise en page et le style.

Préparation

Pour l’utilisation de Bootstrap vous devez télécharger les fichiers sur le site du framework. Créez un dossier public/assets/css et placez-y ces fichiers que vous récupérez dans l’archive téléchargée :

Création de la page

Voici ce que je vous propose :

Le template

A partir du code HTML nous allons insérer le code PHP pour rendre notre page dynamique. Essentiellement nous voulons adapter le menu et le contenu :

?
<!DOCTYPE html>
<html lang="fr">
<head>
<meta charset="utf-8">
<title>Mon beau site</title>
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link href="assets/css/bootstrap.min.css" rel="stylesheet" type="text/css">
<link href="assets/css/bootstrap-responsive.min.css" rel="stylesheet">
<link href="assets/css/main.css" rel="stylesheet" type="text/css">
<!-- HTML5 shim, for IE6-8 support of HTML5 elements -->
<!--[if lt IE 9]>
      <script src="http://html5shim.googlecode.com/svn/trunk/html5.js"></script>
<![endif]-->
</head>
 
<body>
<div class="container">
  <header class="row">
    <div class="span12">
      <div class="hero-unit">
        <h1>Mon beau site !</h1>
      </div>
    </div>
  </header>
  <nav class="row">
    <div class="navbar span12">
      <div class="navbar-inner"> <a class="btn btn-navbar" data-toggle="collapse" data-target=".nav-collapse"> <span class="icon-bar"></span> <span class="icon-bar"></span> <span class="icon-bar"></span> </a>
        <div class="nav-collapse collapse">
          <ul class="nav">
          <?php
            for($i = 0; $i < count($data['menu']); ++$i) {
              echo '<li';
              if($i == $data['page']) echo ' class="active"';
              echo '><a href="'.$i.'">'.$data['menu'][$i].'</a></li>';
            }
          ?>
          </ul>
        </div>
      </div>
    </div>
  </nav>
  <section class="row">
    <div class="span12">
      <?php echo $contenu; ?>
    </div>
  </section>
  <hr>
  <footer id="bas"> © Mon beau site... </footer>
</div>
</body>
</html>

Cette construction suppose qu’on reçoit un tableau $data qui contient un autre tableau $menu avec le nom des pages et une variable $contenu avec le contenu de la page. Enregistrez ce fichier en le nommant template dans le dossier app/views.

La route

Il ne nous reste plus qu’à écrire la route pour faire fonctionner tout ça :

?
Route::get('/{page?}', function($page = 0)
{
    $data = array(
        'menu' => array('Accueil','Page1','Page2','Page3'),
        'page' => $page
    );
    return View::make('template')->with('data', $data)->nest('contenu', 'page'.$page);
})->where('page', '[0-3]');
 
App::missing(function($exception)
{
    return 'Oups ! Je ne connais pas cette page !';
});

La syntaxe est simple et claire et ça fonctionne très bien .

On attend le numéro de page (avec 0 comme valeur par défaut), on définit les données à transmettre au template et on contraint le numéro de page avec la clause « where ».

Enfin j’ai prévu un message pour signaler que la page n’existe pas, nous n’avons pas encore vu les messages d’erreurs mais au moins l’application est ainsi complète et fonctionnelle.
